---
name: venom-premium-ui-qa
description: review Venom ui quality before declaring a task done, especially when a change affects layouts, components, calls to action, support flows, theme parity, accessibility, or perceived premium quality.
---


    Read `../_shared/premium-checklist.md` first.
    Read `../_shared/venom-context.md` when the change touches templates or CSS.

    Review in this order:
    1. design-system compliance,
    2. theme parity,
    3. ux honesty,
    4. accessibility,
    5. responsiveness.

    Return:
    - blockers,
    - warnings,
    - exact file areas likely causing the issue,
    - a concise ship/no-ship verdict.

    Keep the review concrete and tied to changed UI, not generic theory.
