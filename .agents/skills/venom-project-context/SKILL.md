---
name: venom-project-context
description: load Venom project context before coding, reviewing, refactoring, or planning when the task needs the product background, canonical file locations, design-system constraints, or known recurring failures.
---


    Read `../_shared/venom-context.md` first.
    Read `../_shared/prompt-patterns.md` when deciding how aggressive or lightweight to be.

    Use this skill to ground the work before touching code.

    Workflow:
    1. Summarize the task in Venom terms.
    2. Identify the smallest relevant file zone.
    3. List the applicable non-negotiables.
    4. Continue with the main task using the lightest possible workflow.

    Output:
    - a short context snapshot,
    - the file zone to inspect,
    - the top risks to avoid.
